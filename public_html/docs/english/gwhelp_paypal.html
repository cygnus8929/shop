glFusion Shop Plugin Configuration

Main Business Email
This is the "Regular" account e-mail for PayPal. If you have only one account, this must contain it even if it is a "micro-payment" account.
Micro-Payment Email
This is your "Micro-payment" account e-mail, if you have one. If you don't have a micro-payment account, this should contain you regular account email.
Testing (Sandbox) Mode?
Check this box during testing. The Sandbox URL will be used instead of the Production URL.
Micro-Payment Threshold
Enter the threshold amount for micro-payments. Order amounts below this amount will be charged using your micro-payment account e-mail, orders totaling this amount or higher are charged using your regular e-mail.
Encrypt PayPal Buttons?
Check this box if you have set up the necessary certificates to use PayPal Encrypted Buttons. "Buy Now" buttons are encrypted and cached with the product; checkout buttons are encrypted on the fly. If encryption fails, regular HTML buttons are used. Encryption requires several steps to set up, so consult PayPal's documentation.
Paypal Certificate File
Enter the complete path to the public SSL certificate file that you've downloaded from Paypal. This file must be readable by your web server.
Paypal Certificate ID
Enter the certificate ID created by PayPal and assigned to your public key. This is required so that PayPal can decrypt your encrypted button data.
Micro-Payment Certificate ID
If you use micro-payments, you must upload your public certificate to both accounts. Enter the certificate ID created by PayPal for your micro-payment account.
Private Key File
Enter the complete path to your private SSL key file. This file must be readable by your web server and should not be placed in a publically-accessible directory. The security of your encrypted PayPal buttons depends upon this key remaining private.
Public Key File
Enter the complete path to the public SSL key file that you've provided to Paypal. This file must be readable by your web server and should not be placed in a publically-accessible directory, despite its name.
PayPal URL
Enter the PayPal URL, e.g. "https://www.paypal.com" (for production).
Enabled
Check this box to indicate that this gateway is enabled for use. If not checked, the PayPal gateway will not be used.
Buttons

Select the buttons that are supported by this gateway. This is primarily meant to reduce clutter in blocks and product lists, where you may want only one gateway handling immediate purchases. In some cases, a gateway may not be able to handle a type of service, such as donations.

If this is the only gateway in use, then all button types should be checked.

API Username
Enter the API username associated with your application at Paypal. Required to use the Net Terms payment gateway.
API Password
Enter the API password for your application. Required to use the Net Terms payment gateway.
Authorized Group
Select the glFusion group that is allowed to use this gateway. You may wish to restrict access to some gateways sucn as terms or C.O.D.
Order
Enter a number to represent the priority of this gateway (lower number means higher priority).
Enable Donations?
Check to enable donations via Paypal. Only certain types of organizations are eligible to receive donations. If unchecked, a normal "Buy Now" button is used.
